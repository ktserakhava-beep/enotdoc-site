# Track 02 - ROP

## Role
- Second discovery/software track

## Default ICP
- Importer or distributor in Belarus
- 1–3 legal entities
- 100–1000 SKU
- ROP process lives in Excel / 1C / source documents

## Secondary ICP
- Producer / importer of packaging with stable quarterly volume and clear internal owner

## Qualification
- Quarterly recurring cycle is real
- Process already hurts due to Excel / 1C / fragmented source docs
- One clear owner or small owner group exists
- Company is ready to share initial source docs for pilot
- SKU and packaging complexity is meaningful, but not extreme

## Disqualification
- 4+ legal entities with fragmented systems at the start
- Tens of thousands of SKU without any master data
- No clear owner of process
- No readiness to share documents
- Strong dependence on heavy ERP project before any result

## Default Offer
- ROP setup + first-quarter support
- Data collection + packaging mapping + report draft + payment/control logic

## Default Pricing Logic
- Setup + first quarter package

## Default Pricing Test
- Main: fixed setup + first-quarter package for 1 legal entity and agreed SKU/document complexity
- Backup: paid audit / data-map workshop if buyer is cautious

## What To Do Now
- 3–5 discovery calls
- Keep one-pager and script ready
- Capture buyer / owner patterns
- Validate who actually owns packaging data and payment logic
- Test willingness to share source docs

## What Not To Do
- No separate full GTM
- No mini-ERP
- No deep integration project

## Offer Logic
- Sell quarterly operating contour, not "eco-consulting"
- Best pain framing:
  - packaging data scattered across docs
  - hard to classify packaging and bases
  - quarterly deadline pressure
  - risk of overpayment or wrong basis

## Success Signals
- 2+ companies confirm process lives in Excel and hurts
- 1+ company asks for proposal on setup / first quarter
- Buyer openly discusses budget, scope and source docs

## Pause Signals
- Owner of process remains unclear after several calls
- Nobody is ready to share source docs
- All responses reduce issue to "accountant will handle it"

## Anti-Goals
- Do not build SKU master-data monster too early
- Do not position as full ERP/1C project
- Do not launch separate cold GTM before waste traction
