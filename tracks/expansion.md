# Track 05 - Expansion

## Role
- Later expansion modules

## Recommended Order
1. Ecopassport
2. Emissions
3. Water

## Expansion Logic
- Add only when core clients request adjacent document/workflow layers
- Keep scope lightweight

## Activation Triggers
- Ecopassport:
  - 3+ requests per quarter
  - 5+ active accounts with stable waste-core data
- Emissions:
  - 3+ warm accounts with stationary sources and permit/archive pain
  - 2+ ready to pay for document/add-on layer
- Water:
  - 2–3 industrial accounts with existing permits and repeating admin/document pain

## Postpone Signals
- Requests start from engineering, measurements, NDS, treatment plants, exceedances
- Every case is unique and requires field/technical analysis before document layer
- Current core delivery is not yet stable

## Safe Slices
- Ecopassport: annual update workspace
- Emissions: permit/archive/renewal desk
- Water: permit/admin workflow only

## What Not To Do
- No heavy engineering
- No labs and measurements as product core
- No early cold GTM for these modules

## Module Logic
- Ecopassport:
  - best first expansion because it summarizes existing data and supports annual update
- Emissions:
  - add as document and contractor coordination layer, not engineering service
- Water:
  - only add as admin/permit workflow around existing contour, never as first deep module
