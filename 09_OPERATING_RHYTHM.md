# Operating Rhythm

## Purpose
- Keep the project alive through files, not chats
- Make any new chat usable with minimal context loss
- Force important learnings back into project memory

## Default Chat Usage

### Chat 1: Master
Use for:
- strategic decisions
- changing priorities
- reviewing track status
- deciding what to do next

### Chat 2: Execution
Use for:
- outreach
- pipeline
- proposals
- ICP/account work
- call follow-ups

### Chat 3: Temporary Research
Use only if needed for a narrow side question.
If it creates something useful, migrate the conclusion into the project files immediately.

## What To Read At The Start Of Any New Chat
1. `00_MASTER.md`
2. `07_DASHBOARD.md`
3. `08_NEXT_STEPS.md`
4. relevant `tracks/*.md`
5. `04_PIPELINE.csv`

## What To Update After Each Useful Conversation
- create or update note in `meetings/`
- update `04_PIPELINE.csv`
- if pattern changed, update relevant `tracks/*.md`
- if strategy changed, update `01_DECISIONS.md`
- if near-term tasks changed, update `08_NEXT_STEPS.md`

## What To Update After Each Meaningful Learning

### Update `01_DECISIONS.md` if:
- a strategic choice changed
- an ICP was promoted or rejected
- pricing logic changed
- a track moved from hold to active

### Update `02_TRACKS.md` if:
- track status changed
- trigger activated
- KPI changed

### Update `03_VALIDATION.md` if:
- validation threshold changed
- strong market evidence changed confidence

### Update `05_SOURCES.md` if:
- a source is important enough to reuse later

### Update `06_RISKS.md` if:
- a new red flag appears
- a new scope boundary is learned

## Weekly Review Rhythm

### Once Per Week
- review `07_DASHBOARD.md`
- review `08_NEXT_STEPS.md`
- review `04_PIPELINE.csv`
- review signal patterns from calls
- decide whether track status changes

### Questions For Weekly Review
- What is the current core focus?
- Did we get stronger Waste signals?
- Did ROP cross from discovery into build/sell readiness?
- Is Auto still on hold?
- Did any partner channel produce real intros?
- Is any expansion module starting to repeat in active accounts?

## Monthly Review Rhythm
- clean/archive dead leads
- update track priorities if needed
- update strategy only if real evidence changed
- avoid changing direction based on weak/noisy signals

## Rule For New Chats
When starting a new chat, say:
- Read `00_MASTER.md`, `07_DASHBOARD.md`, `08_NEXT_STEPS.md`, the relevant `tracks/*.md` and `04_PIPELINE.csv`, then continue.

## Rule For Closing Old Chats
- If an important conclusion only exists in chat, migrate it first
- Do not trust chat history as permanent memory
- Files are the source of truth

## Anti-Chaos Rules
- Do not keep important insights only in messages
- Do not let side research create new active priorities automatically
- Do not edit strategy without updating `01_DECISIONS.md`
- Do not run more than one core GTM at a time
