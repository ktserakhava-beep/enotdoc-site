# Meetings

## Purpose
- Store notes from calls, intros, objections and follow-ups
- Keep learnings on disk, not in chats

## Naming
- `YYYY-MM-DD-call-company-name.md`
- `YYYY-MM-DD-objections.md`
- `YYYY-MM-DD-partner-call-name.md`

## Minimum Note Structure
- Who was on the call
- Company / segment
- Main pain
- Current process
- Buyer / blocker
- Budget signal
- Objections
- Next step
- Track affected: Waste / ROP / Auto / Partners / Expansion

## Rule
- After every useful call, create a note here and update `04_PIPELINE.csv`
